> **這份文件用途**：給其他 AI / 工程師看的「YT 聊天室互動」**實作前**設計說明，請幫忙找**還缺什麼 / 怎麼補強增強 / 哪裡會出事**。

## 0. 一句話專案背景

一個**「假 24H AI 直播」**節目：兩位 AI 角色（陳柏偉、王于安）用 Claude（Haiku）**不停聊台灣即時新聞**，像 24 小時 live、其實是**預生成內容循環播放**，輸出到 OBS → YouTube 公開直播。內容是「真實新聞當素材、AI 角色詼諧批評/嘲諷時事」的表演。語言：繁體中文（台灣）。

**要新增的功能**：讓 **YouTube 直播聊天室的觀眾留言能跟 AI 主持人互動**（AI 偶爾「回應觀眾」）。

**為什麼難**：聊天室留言 = **敵對的、不可信的流動資料**（prompt injection、釣魚、誹謗誘導、選舉操弄、未成年/個資、平台規避教學…），而 AI 的回覆是**公開播送**、會出事。

---

## 1. 現有系統（已實作、可建構於其上）

直播節目主體已完成且上線，跟本功能相關的現成基礎：

- **內容供應 = batch 預生成 + pool 循環**：背景一次生 12 段對白存池、播放只從池撈（`/api/next_segment`）。每段帶 topic/tone/angle。
- **熱門新聞「即時插隊」機制**：偵測爆紅新聞 → 即時生一段、放 `_live_insert_queue` → 播放時**優先播**（不進 pool）。← **觀眾互動很可能複用這個「臨時插隊」管道**。
- **★ 已有「輸出閘」（兩層、可重用於本功能的 P5）**：
  - L0 `_gate_prefilter`：regex 高風險樣式（侮辱/未證實犯罪指控/臆測收錢）+ 新聞人名鄰近負評偵測（純字串、免費）。
  - L2 `_llm_safety_judge`：命中才升級、用便宜 LLM（Haiku）帶**新聞原文當上下文**做語意判斷，回 `pass/warn/drop`，能分辨「討論已報導案件（放行）」vs「無端指控/辱罵具名真人（擋）」。
  - 已實測：抓得到關鍵字會漏的具名辱罵、也不誤殺正常新聞討論。
- **prompt 端安全規則**：諷刺現象不指控個人、具名真人不掛負評、傷害題先同情、不站隊藍綠/兩岸、不加新聞沒寫的指控。
- **TTS**：edge-tts（台灣男女聲），有熔斷器（聲音掛掉自動演搞笑梗 + 恢復）。
- **跑馬燈 `state.ticker`**：可放「互動倒數 / 快訊」字幕（前端已接）。
- **常駐免責標示**（跑馬燈最前）：「AI 生成內容・非新聞報導・觀點不代表任何真人」。
- **預算守門 `_check_budget`**（日/月上限、超支自動停生成）。
- 後端 FastAPI（Python）、單機、Windows、OBS Browser Source 抓畫面。

> 重點：**輸出閘（P5）、TTS、臨時插隊、跑馬燈、預算守門已經有**，本功能主要要新增的是 **P0–P4 + P6**（讀留言、normalize、分類、選球、安全摘要、無記憶生成、事故處理）。

## 2. 已定案的 pipeline（兩輪 review 後拍板、P0–P6）

```
[留言 / Super Chat / 暱稱 / 貼圖 / 頻道名 … 全部視為 untrusted]
  P0 硬規則：normalization（去零寬字 / homoglyph / 注音 / 編碼還原）→ 黑名單詞 / URL / 長度上限 / 重複字截斷 / 暱稱清洗
  P1 分類：risk = safe | soft_redirect | grey | hard_block；categories[]；每則獨立分類
  P2 選球：unsafe 不入選；SC 只加權、不免審、不必回；每人/每窗/每日上限；語意去重；unsafe 比例過高 → 自動關互動
  P3 安全摘要：raw comment 不進主 AI、只給 viewer_intent（JSON）；禁主 AI 遵循留言指令 / 複述原文
  P4 主生成：Claude（固定人設 + 今日新聞 + 當批留言摘要、無歷史記憶）→ JSON + risk_self_check
  P5 輸出審核（最關鍵）：對「實際口播文字 / 字幕 / overlay」做 final safety check + 金鑰/prompt 洩漏檢查 + TTS sanitizer
  P6 事故處理：unsafe spike → 自動關互動；canned fallback；human override；audit log（含保存期）；replay 移除流程
```

### 三個 reviewer 特別強調、已採納的原則
1. **分類器「只吐標籤」不算真的擋住 injection**（會被混淆字/編碼/分段/角色扮演繞過）→ 要硬規則 normalization 在前。
2. **不能只有 input gate、必須有 output gate**（真正播出去的是 output、它才會出事）→ 已有輸出閘可複用。
3. **主 AI 不該看 raw comment、只看「安全摘要 / intent」**（杜絕 injection 直達主模型）。

### 核心信條
> 留言 = 敵對流動資料；AI 回覆 = 公開播送；**主 AI 不看 raw**；最後一道 gate 審「實際會播出的文字」；一切外部文字（含 RSS 新聞）都走 untrusted pipeline。

## 3. 紅線分四層

| 層 | 處理 | 內容 |
|---|---|---|
| 🔴 HARD BLOCK | 直接擋、不讓主 AI 看、不播、不耍嘴皮 | 仇恨/種族滅絕否認/去人化、自殺自殘/飲食失調方法、兒少性/grooming、人肉個資、暴力武器毒品詐騙駭客教學、選舉操弄假訊息、誹謗未證實指控、詐騙導流、平台規避（「怎麼講不被 YT 抓」）|
| 🟡 SOFT REDIRECT | 可回、但只能安全轉向、禁個案 | 醫療（禁診斷/開藥）、法律（禁個案勝敗/教規避）、投資（禁明牌/喊單）、心理（禁嘲笑/娛樂化）|
| ⚪ GREY | 信心低 / 政治社會敏感 → 強制「太極罐頭」、不給實質立場 | 政治狗哨（「1450」「那個光頭」）、反串、模稜兩可的敏感人物提問 |
| 🟢 NORMAL | 可正常回、控語氣 | 藍綠議題、名人公開風格、新聞梗、AI bug 梗、社會現象、輕度酸民 |

**台灣在地法律紅線**：選罷法（選前 10 日禁發布/引述民調）、刑法 310 誹謗 / 309 公然侮辱、醫師法/律師法/投信顧問法（專業建議）、個資/人肉、著作權（禁朗讀整篇付費新聞 / 唱完整版權歌詞）。

## 4. 幾個關鍵設計決定（已拍板、但想再被檢視）

1. **raw 隔離 + 喜劇校準**：預設主 AI 只拿 `viewer_intent`；但對 `risk=safe` 的留言，**額外傳一段「消毒後的引用片段」**（過 normalization + 嚴格定界符 `<audience_message>…</audience_message>`），讓主持人能拿原句吐槽（喜劇節目的笑點來源）。grey/unsafe 一律 intent-only 或 drop。可開關。
2. **無狀態 / 極短 window**：聊天互動**不回寫角色長期記憶**；主 AI context 每次只 = 固定人設 + 今日新聞 + 當批留言。**觀眾不能訓練主持人人格**（杜絕 persona drift / 溫水煮青蛙）。新聞對話既有的記憶**絕不混入觀眾內容**。
3. **互動內容 ephemeral**：聊天衍生的回應**不得自動進循環 pool**（pool 是會重播的、敵對內容進去會反覆播）。要進 pool 需人工審。
4. **Super Chat 預算防火牆**：SC = 加權、但**不免審、不跳 gate**；硬性速率上限；每人/每窗/每日上限；多筆 SC 打包合併回。
5. **暱稱不直唸**：預設稱「這位朋友 / 觀眾 A」；暱稱也要分類 + 清洗。
6. **時間補白**：回應延遲統一補白（防 timing 反推黑名單）；所有 API 異常 → 本地 canned 變梗，原始 error 永不上畫面。
7. **群體操控防護**：AI 不能說「很多人都在說」當輿情（除非有外部可信來源）、禁「總結大家想法」這種任務。
8. **敏感實體黑名單**：台灣熱門公眾人物 / 企業 / 網紅 → 觸及就「不予評論 / 中立」。

## 5. 上線前必補 10 項（兩輪 review 的結論、缺一不可公開長跑）

1. output gate（審實際播出文字）← **已有、可複用**
2. raw comment 隔離（主 AI 只看 intent）
3. display name sanitizer
4. TTS / 字幕 / overlay 審核（+ 諧音 / SSML / 長度）← 部分有（edge-tts 吃 plain text）
5. Super Chat 預算 + 安全上限
6. 選舉 / 誹謗 / 兒少 / 個資 法務 checklist
7. 互動內容不得自動進循環 pool（ephemeral）
8. unsafe spike 自動降級 / 關互動
9. 外部新聞來源也視為敵對輸入 ← 已部分（輸出閘帶新聞上下文）
10. 主持人人格不可被聊天室長期馴化（無狀態）

## 6. 規劃的實作順序

1. pytchat 讀 YT 聊天 + console 印（零風險）。**＋需要：無 live 也能測 → 假留言注入端點。**
2. P0 normalization + 硬規則 + 暱稱清洗
3. P1 分類器（risk + categories）+ P2 選球（SC 上限 / 去重 / unsafe-ratio 自動關）
4. P3 安全摘要（raw 不進主 AI）→ P4 主生成（無記憶）
5. P5 輸出 gate（複用現成）+ TTS sanitizer
6. P6 事故處理（auto-pause / canned fallback / audit log）
7. 前端：跑馬燈倒數（複用 ticker）+「這位朋友」稱呼
8. **先私人測長跑、確認 P5/P6 穩了再考慮公開**
