# 去背工具 使用說明（char_keyer）

> 把 GPT 生的「綠幕角色人像」一鍵變成可直接放進場景的透明 PNG。
> 對應檔案：根目錄 `去背工具.bat`（拖放用）＋ `tools/char_keyer.py`（實際程式）。

---

## 這個工具做什麼

一張「綠幕背景的角色圖」丟進去 → 自動完成 3 件事：

1. **去綠幕** → 背景變透明（自動偵測綠幕顏色、去綠邊）
2. **清雜點** → 去掉 GPT 圖常有的白色 `+` 對位標記等孤立雜點（只留最大的人物塊）
3. **置中對齊** → 等比縮放、置中貼到 **1254×1254 正方形**，並**對齊現有 `emo_idle.png` 的人物高度與腳底位置**

→ 出來的檔可**直接覆蓋** `assets/char_xiaomei/` 同名檔，場景重新整理就換好（程式、尺寸、站位都不用改）。

---

## 最快用法：拖放

> **把「圖檔」或「整個資料夾」拖到根目錄的 `去背工具.bat` 上放開。**

- 處理好的圖會出現在來源旁邊新建的 **`keyed/` 資料夾**
- 確認 OK → 把 `keyed/` 裡的檔複製進 `assets/char_xiaomei/` 覆蓋同名 → 重新整理

雙擊 `去背工具.bat`（沒拖檔）會印出用法提示。

**輸出檔名 = 輸入檔名。** 所以丟進去前先把 GPT 的圖命名成最終要的名字：

| 你要的表情 | 檔名 |

| 講話 | `emo_talk.png` |
| 微笑 | `emo_smile.png` |
| 思考 | `emo_thinking.png` |
| 驚訝 | `emo_surprised.png` |
| 懷疑 | `emo_skeptical.png` |
| 揮手 | `emo_wave.png` |
| 生氣 | `emo_angry.png` |
| 歡呼 | `emo_cheering.png` |
| 大笑 | `emo_laughing.png` |
| 放鬆 | `emo_relieved.png` |

→ 例如把「講話」那張存成 `emo_talk.png` 丟進去，出來就是 `emo_talk.png`，直接覆蓋即可。

## 批次處理一整套表情（推薦流程）

1. GPT 生全部表情（用同一張全身當 reference、只改表情、眼睛都看鏡頭）
2. 各自命名成上表的檔名
3. 全部丟進**一個資料夾**
4. 把那個**資料夾**拖到 `去背工具.bat` → 一次全部去背＋對齊
5. 把 `keyed/` 裡的全部複製進 `assets/char_xiaomei/` 覆蓋 → 重新整理

## 命令列用法（進階）

```
python tools/char_keyer.py 圖或資料夾 [選項]
```

| 選項 | 說明 |
|---|---|
| `--ref 路徑` | 對齊參考圖（預設 `assets/char_xiaomei/emo_idle.png`）。新圖會對齊這張的人物高度/腳底 → 全套位置一致 |
| `--out 資料夾` | 輸出資料夾（預設：來源旁的 `keyed/`） |
| `--size N` | 畫布邊長（預設 1254） |
| `--scale F` | 手動指定縮放倍率（不自動對高度）；給「舉手過頭」等特殊姿勢用 |
| `--no-clean` | 不清雜點 |

範例：
```
python tools/char_keyer.py C:\Users\Administrator\Downloads\新圖資料夾
python tools/char_keyer.py emo_wave.png --scale 0.93
```

## 常見問題

- **綠幕沒去乾淨 / 有綠邊**：生圖時用**均勻純綠、無漸層**的背景最好。殘留通常是漸層綠或陰影綠造成；可重生一張更乾淨的綠幕底。
- **人物變太小（多半是揮手/歡呼這種舉手過頭的姿勢）**：自動對齊是「把人物外框高度對到參考圖」，手舉過頭會讓外框變高 → 人被縮小。解法：那張改用 `--scale`（給跟別張一樣的倍率），或生圖時手別舉超過頭頂。
- **位置跑掉 / 大小跟別人不一致**：確認 `--ref` 指到正確的 `emo_idle.png`（預設就是它）。所有表情都對齊同一張 ref 就會一致。
- **沒清掉 `+` 標記**：清雜點需要 `scipy`。若沒裝會提示你 `pip install scipy`；裝了就會自動清。
- **輸入要是「綠幕圖」**：這工具是為綠幕背景設計的。若丟進去的圖背景不是綠的（例如已透明、或白底），去背結果會不對。
