# 「假 24H AI 直播」輸出閘（Output Gate）優化與增強方案建議書

這份文件針對「假 24H AI 直播」節目的最後一道防線——輸出閘（Output Gate），提供具體、可實作且符合台灣法律情境的優化與增強方案。在兼顧「低成本、低延遲、高防禦力」的前提下，優化現有的純 Regex 關鍵字比對機制。

---

## 1. 語意級攔截：最務實的第二道防線設計

要在現有的 Regex 基礎上加入 LLM 語意分類器，最務實的作法是**「完全放在生成時（入池前）處理」**，絕對不要放在播放時。播放時必須保持毫秒級的快取攔截，而生成時則允許秒級的深度審查。

### 執行時機與批次策略
* **執行時機**：批次生成對白段（對白池）後，立即送入語意檢查。通過者入池，不通過者直接丟棄或觸發改寫。
* **批次策略**：採用**逐段審查（Segment-based）**。不要將多個主題打包審查，因為只要其中一段違規，整批都必須被污染丟棄。逐段審查可以確保 JSON 結構清晰，模型判斷最為精準。
* **模型選擇**：建議使用極低成本的輕量模型（如 Gemini 1.5 Flash 或 GPT-4o-mini）。由於生成端已經使用了 Claude Haiku，審查端改用不同模型家族，能有效避免同一個模型對安全規則的「認知盲區」。

### 語意審查 Prompt 設計範本
你可以將下列邏輯作為審查模型的系統指令（System Prompt），要求模型以結構化 JSON 輸出以利程式解析：

> **【系統角色】**
> 你是台灣法律與媒體自律審查專家。請評估一段雙人對白是否違反台灣法律（誹謗罪、公然侮辱罪、選罷法意圖使人不當選）。
> 
> **【審查標準】**
> 1. **具名指控**：對白中是否包含真實存在的具體人名（政治人物、公眾人物、官員、藝人等），且對該人進行「未經法庭定讞的犯罪指控」（如貪污、收賄、洗錢、掏空）或「人格貶損的侮辱」。
> 2. **有罪推定**：是否對正在調查中、司法未定讞的案件，直接斷定某個具體真人或單位有罪。
> 3. **無端臆測**：是否在沒有新聞事實依據的情況下，指控特定對象「一定有收錢」、「背後有陰謀」。
> 
> **【排除條件（允許放行）】**
> * 若對白僅是「引述已公開報導的新聞事實」（例如：「新聞報導說某某官員被起訴了」），且角色隨後討論的是「制度漏洞」或「社會現象」，而非直接加碼人身攻擊或人格判定，則視為安全。
> 
> **【輸出格式】**
> 請直接輸出 JSON 格式，嚴禁任何其他解釋文字：
> {
>   "safe": true/false,
>   "reason": "若不安全，請簡述違反哪一條標準，否則留空",
>   "risk_level": "HIGH/MEDIUM/LOW"
> }

### 成本與延遲取捨
* **成本**：以目前輕量模型每百萬 Token 僅約 0.075~0.15 美元的價格計算，審查一段 8 句的對白成本幾乎可以忽略不計。
* **延遲**：會讓批次生成的總時間增加約 1~2 秒，但因為是預先生成入池，對前台 OBS 播放及 TTS 生成**完全零延遲影響**。

## 2. 降低誤殺率：出處與語境感知脈絡機制

純 Regex 會誤殺「正在理性討論新聞事件」的正常對白，是因為它缺乏上下文對照。要讓輸出閘分清「討論已報導案件」與「無端惡意指控」，需要改變資料餵送方式。

### 脈絡對照解決流程
在語意審查時，採取**「新聞原文脈絡 + 生成對白」**並行驗證的機制：
1. **資料綁定**：將該段對白所依據的「新聞標題與摘要（RSS 內容）」作為脈絡（Context），與對白一併餵給審查模型。
2. **歸因檢查（Attribution Check）**：
   * 審查模型會比對對白中的敏感詞（如「貪污」）是否起源於新聞原文事實。
   * **情境 A（允許放行）**：新聞本身就是「某官員涉貪遭起訴」，對白內容為「這群人天天在聊那個涉貪案，到底制度哪裡出問題？」。模型比對後會發現這屬於**評論已報導事實**，判定為安全。
   * **情境 B（攔截丟棄）**：新聞只是「某官員出席慈善活動」，對白卻脫口而出「他看起來就是有貪污」。模型比對後發現新聞並無此記載，屬於**無端指控**，判定為不安全。

## 3. 輕量化具名真人 + 負評偵測

在不部署複雜、高成本的命名實體識別（NER）深度學習模型的前提下，最適合獨立開發者的輕量化做法是**「動態白名單機制 + 距離演算法（Distance Algorithm）」**。

### 具體實作邏輯
1. **動態名單提取**：
   * 每天系統抓取新聞 RSS 時，利用簡單的文本處理或輕量 LLM，順便提取出當天熱門新聞的「核心人名」（通常每批 30 條新聞中，核心公眾人物不超過 20 個）。將這些人名動態加入當天的暫存白名單。
2. **距離限制比對（Regex 升級版）**：
   * 利用 Regex 搭配字元距離限制，抓取人名與負面/侮辱詞彙的鄰近關聯性，而非全段盲目比對。
   * 設定規則：當「特定人名」與「負面詞/侮辱詞」在前後 N 個字元內同時出現時，即觸發攔截。
   * *範例邏輯*：`[人名] 距離 10 個字以內出現 [智障/貪污/騙子]`，或者 `[智障/貪污/騙子] 距離 10 個字以內出現 [人名]`。
3. **優點**：不需跑龐大的 NER 模型，單純用 CPU 進行字串距離計算即可在毫秒內抓出「特定人名周圍出現人格貶損或未證實指控」的高風險組合，極適合放在播放時做最後一刻的複檢。

## 4. 三色燈安全分級與對白改寫策略

目前的二元處理（命中就整段丟棄）雖然最安全，但會造成生成 Token 的浪費與對白池產能不足。改採**「三色燈安全分級機制」**能大幅提升對白池的利用率。

### 分級架構與處理策略

| 風險等級 | 判定標準 | 處理動作 | 執行策略說明 |
| :--- | :--- | :--- | :--- |
| **紅燈 (High Risk)** | 出現極端字眼、無端犯罪指控、直呼真人名字痛罵、涉及傷亡案件娛樂化。 | **直接丟棄** | 寧可誤殺，絕對不播，後台自動補生新段。 |
| **黃燈 (Medium Risk)**| 字眼稍微踩線（如使用模糊諷刺詞）、評論尺度邊緣，但未具名真人。 | **安全改寫 (Rewrite)** | 不丟棄，送回輕量模型進行局部詞彙安全替換。 |
| **綠燈 (Low Risk)** | 完全符合規範，純粹探討體制現象、自嘲或幽默諷刺。 | **直接入池** | 順暢進入對白池播放。 |

### 如何改寫而不破壞對白連貫性？
當段落被判定為黃燈時，不要只做單一死板的詞彙替換（容易造成前後文法崩壞），而是將**整段對白與違規原因**送回 Haiku 或審查模型，並給予改寫指令：
> 「這段對白因出現『一定是拿了錢』而觸發黃燈風險。請在**不改變兩位角色（陳柏偉、王于安）的幽默語氣、個性與前後對話邏輯**的前提下，將該敏感涉嫌詞彙改寫為合法的諷刺性言論（例如改為：『這背後的利益關係，真是讓人霧裡看花』）。請直接輸出改寫後的對白結構。」

## 5. 詞表治理與台灣法律實務詞庫

單靠社群網路通用的髒話庫是不夠的，因為直播面對的是台灣刑法。詞表必須與台灣司法院的判決實務接軌，並進行配置分離。

### 台灣妨害名譽與政治嘲諷詞庫建議
1. **對接司法院刑事判決實務高頻詞**：
   * 根據台灣法院判決妨害名譽（公然侮辱）成立的實務案例，許多日常詞彙極易踩線。應將以下常被忽略的公然侮辱成罪詞彙納入基礎詞庫：`奴才、垃圾、神棍、不要臉、神經病、賺差價、神豬、下流`。
2. **納入台灣社群與政治鄉民變體詞（動態維護）**：
   * 台灣網路政治鄉民常使用諧音或變體詞來規避字面審查（如：`1450、割韭菜、綠畜、藍白拖、塔綠班、死忠的、網軍`）。
   * 這類詞彙雖不一定百分之百觸犯誹謗罪，但極易引發直播間觀眾洗頻、對立或惡意檢舉，應列入**黃燈標籤**，限制其在對白中出現的頻率。
3. **配置分離治理**：
   * 嚴禁將關鍵字寫死在 Python 程式碼中。應將敏感詞庫抽離至獨立的 `json` 或 `yaml` 設定檔中。
   * 開發一個簡單的內部管理機制，每當台灣有新時事（例如新的政治流行語或爭議事件）發生時，直接更新設定檔即可即時生效，免去改動與重新部署核心程式碼的風險。

## 6. 全局整體安全管線架構增強

以「24H 不停生成 + 公開播送 + 成本敏感」為前提，現有的多層管線（Prompt → 品質黑名單 → 輸出閘）架構完整，但在長期運作上還缺乏以下兩道自我防護與應變防線：

### 防線補強一：反饋迴圈（Feedback Loop）與統計日誌
* **現狀問題**：輸出閘攔截了不安全的內容，但前端生成 Prompt 並不知道自己觸犯規則，導致後續持續生成類似的違規內容，造成 Token 成本的極大浪費。
* **增強做法**：建立一個**自動阻斷機制**。如果某個新聞主題在輸出閘連續被攔截超過 3 次（例如因為涉及高度敏感的政治人物、複雜的司法官司），系統應自動將該新聞主題/URL 加入當天的「生成黑名單」，今天內不再碰這個話題，從源頭停損。

### 防線補強二：即時「緊急中斷與消音閘」（Emergency Break）全局開關
* **現狀問題**：過濾都在 TTS 生成前。如果直播進行到一半，台灣突然發生重大天災（如大地震、強烈颱風災情）或重大傷亡事故，舊的對白池裡可能還殘留著幾小時前生成的「調侃、揶揄」某些政策或現象的內容。此時播出會引發極大的公關災難與社會反感。
* **增強做法**：在播放時的最後一道輸出閘（`/api/next_segment`）最末端，加上一個**「全域安全模式切換開關」**。
* 當台灣發生重大悲劇時，後台一鍵啟動「極度保守/悲傷模式」，輸出閘會瞬間過濾掉所有帶有諷刺、嘲諷、調侃、爭論語氣（`tone: mocking / debate / critical`）的對白，只允許播出溫和、科普、或純粹宣導性質的段落，或者自動切換為播放「防災/祈福常規文案」，直到事件平息，確保直播絕對不翻車。

## 相關資訊來源與參考

* **中華民國刑法（第 309 條公然侮辱罪、第 310 條誹謗罪）**：關於「真實惡意原則」、「對於可受公評之事為適當之評論」之法律阻卻違法事由定義。詳細法條可查閱《法務部全國法規資料庫》。
* **司法院法學資料檢索系統**：可透過刑事判決書查詢妨害名譽中「侮辱」之實務認定詞彙，用以動態更新高風險 Regex 詞表。
* **AI 護欄架構設計（Guardrails for AI）**：參考業界標準 AI 安全框架（如 NeMo Guardrails），其核心核心邏輯皆採用「輸入防護（Input Guardrails） → 核心管線 → 輸出防護（Output Guardrails）」之多層縱深防禦模型。
