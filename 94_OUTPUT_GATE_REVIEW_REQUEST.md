# 94 — 輸出閘（Output Gate）結構說明 + 請求補強/增強

> **這份文件用途**：給其他 AI / 工程師看的「輸出閘」現況說明，請幫忙找**補強或增強的方式**。
> 文件自帶背景，不需要原始碼也能理解。最後一節是想請教的問題。

---

## 0. 一句話專案背景

一個**「假 24H AI 直播」**節目：兩位 AI 角色（陳柏偉、王于安）用 Claude（Haiku）**不停聊台灣即時新聞**，像 24 小時 live、其實是**預生成內容循環播放**，輸出到 OBS → YouTube 公開直播。內容是「真實新聞當素材、AI 角色詼諧批評/嘲諷時事」的表演。語言：繁體中文（台灣）。

**法律情境（台灣）**：誹謗罪（刑法 310）、公然侮辱罪（刑法 309）、選罷法（選舉期間對候選人散布不實）、個資法。所以「AI 講出來的話」= 公開播送，必須避免**罵到具名真人 / 未證實犯罪指控 / 有罪推定**。

---

## 1. 內容生成 + 安全的整體管線

```
新聞 RSS（Google/Yahoo TW、~30 條/批、含分類）
        ↓
Claude Haiku 批次生成「對白段」
   - 每段 = 一個 topic + tone + angle、7~8 句、兩位角色交替
   - prompt 內含安全規則（見第 2 節 A/C + 諷刺現象不指控個人…）
        ↓
品質檢查 _quality_check_dialogue（黑名單擋極端內容：性暴力 / 對未成年暴力）
        ↓
口頭禪過濾 _strip_3q（移除角色口頭禪「3Q」）
        ↓
★ 輸出閘 _output_gate_segment（本文件主角、見第 3 節）→ 不通過整段丟棄
        ↓
存進 pool（一池對白循環播放）
        ↓
播放時（/api/next_segment）：再過一次輸出閘（擋「閘上線前」的舊段）→ 生 TTS → 播
```

**重點**：安全是「多層」的。輸出閘是**最後一道、檢查「實際要播出的文字」**的字面防線。它前面還有 prompt 規則（從源頭防）+ 品質黑名單。

---

## 2. prompt 端的安全規則（源頭防線、給你理解上下文）

寫在生成 prompt 裡（叫模型「別這樣寫」）：

- **A｜具名真人紅線**：topic/新聞含真實人名（政治人物、官員、藝人、企業負責人）→ 只談「事件 / 政策 / 現象」，**負面評價、人格判斷、嘲諷不可掛到人名上**；要批評就把主詞換成「這件事 / 這個政策 / 這種現象」。
- **C｜死亡/傷亡案件**：不可娛樂化、不可當笑點 → 先同情承認嚴重，再把火力對準「制度 / 結構」失靈、不對受害者。
- 其他：諷刺「現象 / 結構 / 規律」不針對個人或政黨；不加新聞沒寫的指控/陰謀論；不對未定讞案件有罪推定；不用「聽說/據傳」包裝指控；不在兩岸/統獨/藍綠站隊。

> A/C 是「叫模型別生成」。但模型不一定 100% 聽話 → 所以需要輸出閘（D）這層**事後攔截**。

## 3. ★ 輸出閘現況實作（本文件重點）

### 3.1 它做什麼
對「**一整段對白的實際文字**」做最後檢查。命中高風險樣式 → **整段丟棄**（寧可少一段、不冒風險；pool 會自動補生新段）。

### 3.2 實作（Python、純 regex 關鍵字比對）

```python
# 高風險樣式（regex, 繁中）。命中任一 → 整段不通過。偏保守、寧可誤殺。
_GATE_PATTERNS = [
    # ① 侮辱性字眼（公然侮辱罪）
    (r'智障|腦殘|白痴|低能|廢物|垃圾人|王八蛋|去死|滾蛋|渣男|賤貨|婊|腦袋裝屎', "侮辱字眼"),
    # ② 未證實犯罪指控（誹謗罪）— 把人/單位講成涉案
    (r'貪污|收賄|賄賂|洗錢|圖利|掏空|賣國|通敵', "未證實犯罪指控"),
    # ③ 臆測收錢 / 動機（陰謀論）
    (r'一定是.{0,6}(收|拿|A)了?錢|根本就是.{0,4}(收|拿)錢|背後一定有', "臆測動機"),
]

def _output_gate_segment(lines):
    """lines = [{"speaker":"aming"/"xiaomei", "text":"...", "emotions":[...]}, ...]
    回 (是否通過: bool, 命中原因: str)。"""
    joined = " ".join(line["text"] for line in lines)   # 整段文字串起來
    for pattern, label in _GATE_PATTERNS:
        if re.search(pattern, joined):
            return False, label        # 命中 → 不通過
    return True, ""                     # 都沒命中 → 通過
```

### 3.3 執行時機（兩個點）
1. **生成時**：批次生成後、存進 pool 前。不通過 → 不入池。
2. **播放時**：從 pool 撈出要播的那段、生 TTS 前再檢查一次（攔截「閘上線前就生成的舊段」）。不通過 → 跳過、改撈下一段（最多重試 4 次）。

### 3.4 一段「對白段」長什麼樣（資料結構）
```json
{
  "dialogue_id": "uuid",
  "topic": "某新聞標題",
  "tone": "react / critical / debate / mocking / ...",
  "angle": "切入角度",
  "lines": [
    {"speaker": "aming",   "text": "陳柏偉講的話…", "emotions": ["talk"]},
    {"speaker": "xiaomei", "text": "王于安接的話…", "emotions": ["skeptical"]}
  ]
}
```

## 4. 已知限制（請針對這些找補強）

1. **只認「字」、不懂「意思」**：
   - **會漏**：用沒列到的詞罵人仍放行，例：「賴清德根本是個無能的騙子」（「騙子/無能」不在名單）→ 漏掉（其實是罵具名真人）。
   - **會誤殺**：新聞本身就在講貪污案，討論「這個貪污案的制度漏洞」→ 命中「貪污」被丟掉（但討論已報導/起訴的案子並非誹謗）。
2. **沒有具名實體辨識（NER）**：無法泛化偵測「具名人 + 負面評價」這個組合（目前靠 prompt A 防、閘只補字面紅旗）。
3. **二元處理**：命中就整段丟，沒有「只改那一句 / 改寫成安全版本 / 分級嚴重度」。
4. **沒有語境/出處感知**：分不出「引述新聞原文」「反諷」「把話安到別人嘴上」vs「角色自己的指控」。
5. **單語**：只處理繁中、固定詞表，名單靠手動維護、容易過時。
6. **規模**：目前用量小（觀眾少）。希望方案能「現在低成本、之後好擴充」。
