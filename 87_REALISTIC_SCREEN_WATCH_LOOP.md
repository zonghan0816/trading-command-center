## 0. 一句話

把 TDT 從「兩人 + 漂浮泡泡、即時生成有空檔、無聲」升級成
**「真人半身主持人 → 轉頭看螢幕(遮住生成延遲) → 轉回開聊 + 語音」的循環表演**，
同時**順帶解決直式 Shorts 的空洞問題**。

---

## 1. 痛點 / 動機

| 現況問題 | 這個設計怎麼解 |
|---|---|
| **沒有語音、看直播覺得「沒效果」**（使用者最大心頭大患）| 加 TTS，轉頭回來時用語音講 |
| 即時生成有延遲 → 換題/換輪有空檔（舊 gap 問題）| **「轉頭看螢幕」把空檔變成劇情**，延遲期間偷偷生成 |
| 直式 Shorts：16:9 塞進 9:16，主角小、上下空洞 | 每階段有單一主角填滿畫面（螢幕 or 主播）→ 裁切不空洞 |
| 動漫感想升級真實感 | 真人半身 PNG（GPT 生圖）|

---

## 2. 核心循環（狀態機）

```
🅰 IDLE_CHAT  平常閒聊（2人面對觀眾、輕鬆對話）
   └─ 背景偷偷做事：搜尋下一則新聞 + 抓圖（或 fallback 底圖）
        ↓
🅱 ANNOUNCE   主播報題：「來，看下一則蝦到爆的新聞」
        ↓
🅲 WATCH_SCREEN  兩人「轉頭看螢幕」（螢幕顯示 新聞圖 + 標題）
   └─ 背景偷偷做事：生成這則的對話 + 生成語音(TTS)
        ↓
🅳 DISCUSS    轉頭回觀眾、開始聊（大泡泡 +／或 語音）
   └─ 背景偷偷做事：準備「下一則」的新聞 / 對話 / 語音 / 圖
        ↓
   🔁 回 🅰，反覆循環
```

### 設計精華（使用者的洞察）

- **「轉頭看螢幕」不是在掩飾延遲，它本身就是真實感**——真主播也會看提詞/資料畫面。
- **遮罩時間 = 免費的生成時間**。看螢幕那幾秒，剛好夠後端搜新聞、生對話、生語音。
- **重複的內容不重生 = 不花錢**——這正是 24H MVP 的「pool 循環」核心，使用者獨立想到。
- **就算遮罩期間短暫空畫面也沒差**，幾秒內可完成搜題/抓圖。

---

## 3. 為什麼這同時解決「直式 Shorts」

每個階段**都有單一主角填滿畫面**：

| 階段 | 直式裁切主角 |
|---|---|
| 🅲 看螢幕 | **螢幕**（新聞圖+標題）占滿 |
| 🅳 開聊 | **講話的主播**占滿 |

→ Shorts 自動裁到「當下主角」，永遠不空洞。等於 TikTok 新聞號的剪法，但**原生就有焦點切換**，不用事後硬裁。

## 4. 可行性結論（使用者三個提問）

### 🔊 語音 TTS — ✅ 能，最高優先，CP 值最高
- 選項：**Edge-TTS（免費）** / OpenAI TTS / Google / Azure / ElevenLabs（最像真人、較貴）
- 台灣中文（zh-TW）：Azure / Google 有不錯台灣腔；ElevenLabs 多語
- 成本低、可行性高 → **建議第一個做**
- 待決定：用哪家、兩位主持人各配什麼聲線（男/女、語速、語氣）

### 🖼️ 自動生圖 — ✅ 能，但有條件
- Claude **不生圖**，要接 **OpenAI 生圖 / Google Imagen / Flux** 等
- 成本：一張約 NT$0.3~1.5，比文字貴；**重複快取重用就不花**
- ⚠️ **真新聞配 AI 生成假圖 = 假新聞風險**。策略：
  - 真新聞 → 抓原文 `og:image` 真圖
  - 抓不到 → **風格化/抽象底圖**（不誤導、不假裝現場）
  - 不要用 AI 生「假裝是新聞現場」的圖

### 🎬 影片生成 — ⚠️ 技術可行但 24H 不划算
- Runway / Veo / Sora：每秒要錢 + 生成要等，跟「AI 影片主播」同樣的成本問題
- 24H 連續用會爆預算 → **特殊橋段偶爾用 / 現階段跳過**
