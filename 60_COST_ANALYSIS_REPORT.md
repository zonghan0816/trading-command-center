# Phase 3 成本分析報告 — Anthropic API 月成本與優化方案

**狀態：** 分析報告（非實作 BRIEF）
**起因：** 使用者考慮上架 YouTube、詢問實際運作成本。發現 24/7 月成本可能達 NT$57,000、需評估優化方向
**結論：** 24/7 直播確實貴、但有 4 個明確降本方向、優化後可降 80%+（NT$57k → ~NT$10k/月）

---

## 一、現況單次成本

### `/api/chat` 單次

| 項 | 預估 token | 單價（USD/M tok）| 成本 USD |
|---|---:|---:|---:|
| Input prompt（含 Step 6.3 反重複 block）| ~3000 | $1.00 | $0.003 |
| Output（3-4 句對白）| ~500 | $5.00 | $0.0025 |
| **單次合計** | — | — | **~$0.0055 ≈ NT$0.17** |

### Step 6.5 prefetch 後：一輪 = 2 次

→ **NT$0.34 / 視覺輪**

### 視覺輪頻率（Step 6.5 後）

- 一輪播放 ~12-15 秒
- next dialogue gap 350ms
- ≈ 每分鐘 4 輪
- ≈ 每小時 240 輪

## 三、為什麼這麼貴

**因為現在的設計是「不停產**」**對話**：

- Step 6.5 prefetch 把頻率翻倍（NT$/小時 ×2）
- Step 6.3 反重複 block 把 input token 撐到 ~3000（NT$/次 +30%）
- 沒有 prompt caching、每次都付完整 input 費用
- 沒有「節目時段」概念、24/7 不停

對「個人練習」很貴、對「商業節目製作」其實便宜（請真人主持人、編劇、剪輯月薪都遠超這個）。

## 四、4 個優化方向（可組合）

### A. Prompt Caching（**最高 ROI**）

Anthropic 支援 prompt caching、固定部分 input 收 **10%** 的價（其餘 input 仍 100%）。

我們的 prompt 結構：

| 區段 | 是否固定 | 預估 token |
|---|---|---|
| 主持人設定（_CHARS）| ✅ 固定 | ~200 |
| Tone structure 說明（8 個）| ✅ 固定 | ~400 |
| 8 個 angle notes | ✅ 固定 | ~300 |
| 字數規則、風格、禁用清單 | ✅ 固定 | ~400 |
| 輸出格式 | ✅ 固定 | ~50 |
| 話題區塊（topic / summary / keywords）| ❌ 動態 | ~150 |
| Angle block | ❌ 動態 | ~80 |
| Anti-repeat block | ❌ 動態 | ~400 |
| **固定 / 動態比** | **~75% / 25%** | ~1350 / ~630 |

→ 固定部分 1350 tokens × $1/M × 10% = $0.000135（原 $0.00135、省 90%）
→ Input 整體從 $0.003 降到 ~$0.00077

**預期效果**：每次降 ~50%、月成本 **NT$57k → NT$28-30k**

**改動範圍**：
- `server.py` `_build_prompt` 拆固定/動態區段
- Anthropic SDK 加 `cache_control` 參數
- ~30 行改動、不影響行為

**ROI**：⭐⭐⭐⭐⭐（改動小、省最多、不影響觀眾體驗）

### B. 關 Step 6.5 prefetch

把每輪 2 次 API 變回 1 次。

**預期效果**：月再砍 50% → **NT$14-15k**（疊在 A 之上）

**代價**：對話 gap 從 0.5s 變回 5~10s（Step 6.5 之前的狀態）

**改動範圍**：OfficeScene.js 拿掉 `_prefetchNextDialogue` 觸發、改回單 fetch 路徑

**ROI**：⭐⭐⭐（視覺體驗變差、但省一半）

### C. 拉長 next dialogue gap

從 350ms 改成 30-60 秒「節目換段時間」。

每小時輪數從 240 砍到 60-120。

**預期效果**：月再砍 50-66% → **NT$5-7k**（疊在 A+B 之上）

**代價**：節目節奏變慢（適合「思考型節目」、不適合「快嘴鄉民」）

**改動範圍**：OfficeScene.js 一行 delayedCall 數字

**ROI**：⭐⭐⭐⭐（改一行、省很多、看節目定位）

### D. 換模型

| 模型 | 輸入 / 輸出價 | vs Haiku 4.5 |
|---|---|---|
| Claude Haiku 4.5（現在）| $1 / $5 | baseline |
| OpenAI gpt-4o-mini | $0.15 / $0.6 | **省 80-90%** |
| 本地 Llama 3.1 8B | 電費而已 | 省 95%+ 但需顯卡 |
| Google Gemini Flash | $0.075 / $0.3 | 省 90%+ |

**預期效果**：月成本 **NT$5k 以下**（疊在 A+B+C 之上）

**代價**：
- Claude 對中文鄉民語氣的掌握目前是強項、換模型可能對白品質下降
- 各家 SDK 要重寫整合
- Prompt 要重調適應新模型

**改動範圍**：大、整套 LLM 接線重寫

**ROI**：⭐⭐（最便宜但風險最高、品質可能崩盤）

## 五、推薦組合（依使用情境）

### 情境 1：偶爾測試 / 錄影上傳

**不用優化**。10 分鐘 NT$14、隨便玩。

### 情境 2：每天 1-2 小時固定時段

**做 A**。預期月成本 NT$2,000-3,000。改動小、品質不變。

### 情境 3：每天 8-12 小時內容流

**做 A + C**。月成本 NT$2,000-3,000。節奏放慢、保留 prefetch 給快節奏關鍵段。

### 情境 4：真要 24/7 直播當頻道特色

**做 A + B + C**。月成本 NT$5,000-7,000、可承受。
- 若還要更省：做 D（換模型）、但品質風險自負

### 情境 5：先試水 1-2 個月看流量

**只做 A**（最低改動投入）+ 工作時段限定播放（手動開關 server）。
- 月成本 NT$5,000-8,000

## 八、Anthropic prompt caching 技術細節（給實作參考）

需要：
- Anthropic SDK 1.x（Python anthropic 套件、已用）
- 把 prompt 拆 system + user
- 加 `cache_control` 參數標記哪些段要 cache

範例：

```python
msg = await client.messages.create(
    model="claude-haiku-4-5-20251001",
    max_tokens=800,
    system=[
        {
            "type": "text",
            "text": FIXED_SYSTEM_PROMPT,  # 主持人設定 + tone + angle notes + 規則
            "cache_control": {"type": "ephemeral"},  # ← 標記要 cache
        }
    ],
    messages=[
        {
            "role": "user",
            "content": dynamic_topic_block + angle_block + anti_repeat_block,
        }
    ],
)
```

Cache 規則：
- Ephemeral cache：5 分鐘失效
- 24/7 直播每 5 分鐘 reuse 一次、命中率 ~100%
- 第一次仍付完整 input 費用（cache write 是 1.25 倍價）、之後 cache read 是 0.1 倍價

實際省幅：

| 場景 | 第一次 | 之後（cache hit）|
|---|---|---|
| 不用 cache | $0.003 | $0.003 |
| 用 cache | $0.0038（+25%）| $0.0003（-90%）|

5 分鐘內第 2 次以後就賺、24/7 跑均下來省 ~85% input 成本。

## 十、結語

**「24/7 月 NT$57k」是真的、但前提是「沒做任何優化」**。

對個人專案：
- 偶爾測試完全沒問題（10 分鐘 NT$14）
- 每天 1-2 小時也可承受（月 NT$2-3k）
- 24/7 必須先做 prompt caching（A）、否則會月燒上萬

對未來方向：
- 若 YouTube 流量起來、考慮 monetization
- A + C 是甜蜜點、月 NT$10k 內、品質不掉
- D 是最後手段、品質風險自負

請 GPT 裁示是否要做 A（prompt caching）、或先觀察使用模式再決定。
